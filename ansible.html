6. Ansible による自動化

複数ノードの初期構築や保守作業を自動化するために、Ansibleを導入しました。

主な用途と実行内容

  • 各ノード（pipf2、pipf5AIなど）のパッケージアップデート
  • 定期リブートやcron設定の統一化
  • sshd_configやfirewalldの自動反映

実行したPlaybookの例

- hosts: all
  become: yes
  tasks:
    - name: パッケージのアップデート
      apt:
        update_cache: yes
        upgrade: dist

    - name: 再起動のスケジュール登録
      cron:
        name: "weekly reboot"
        minute: 0
        hour: 3
        weekday: 0
        job: "/sbin/reboot"
